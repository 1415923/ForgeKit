# 版本推进闸门检查清单

用于决定是否允许进入下一个大版本。

## 进入第一版开发前

- [ ] `docs/项目开发方案.md` 已完成第一版。
- [ ] `docs/版本路线图.md` 已完成第一版。
- [ ] 技术栈选择已有理由和替代方案。
- [ ] 软硬件落地条件已确认或列出风险。
- [ ] v0.1.0 范围明确。
- [ ] v0.1.1 review/refactor gate 已规划。

## 大版本结束后

- [ ] 当前大版本功能已完成或明确裁剪。
- [ ] 当前大版本 EPIC / FEAT / TASK / BUG 状态已闭合。
- [ ] 版本更新记录已同步。
- [ ] 架构设计已同步。
- [ ] 接口、数据库、部署、测试文档已同步。
- [ ] 环境矩阵和发布流水线已同步。
- [ ] 已检查重复代码和重复文件。
- [ ] 已检查模块边界和目录结构。
- [ ] 已检查 Critical 或 Unknown 代码区域的所有权和评审结论。
- [ ] 已检查是否需要抽象、合并、复用。
- [ ] 已检查测试和验证是否可接受。
- [ ] 已检查安全、权限、密钥、外部动作。
- [ ] `docs/技术债记录.md` 已更新。
- [ ] `docs/质量指标记录.md` 已更新。
- [ ] 严重事故或重复缺陷已更新 `docs/事故复盘.md` 或 `docs/缺陷复盘.md`。

## 允许进入下一个大版本前

- [ ] review/refactor 中版本已完成。
- [ ] 中版本结论写入 `docs/版本路线图.md` 或 `docs/版本更新记录.md`。
- [ ] 技术债变化已记录。
- [ ] 质量指标变化已记录。
- [ ] SEV-1 / SEV-2 行动项已关闭或明确接受。
- [ ] S2/S3 安全风险已关闭或明确接受。
- [ ] 发布、部署和环境治理缺口已关闭或进入后续计划。
- [ ] 代码所有权和评审责任缺口已关闭或进入后续计划。
- [ ] 项目任务模型中未完成事项已延期、裁剪、阻塞或进入后续计划。
- [ ] 未完成事项已进入后续计划。
- [ ] 用户明确确认允许推进。

如果以上条件不满足，Codex 应拒绝直接推进下一个大版本，并提醒人工确认。
