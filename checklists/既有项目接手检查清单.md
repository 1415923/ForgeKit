# 既有项目接手检查清单

用于接手已有项目、先审计修 bug、再做后续开发。

## 接手前

- [ ] 明确项目来源、业务用途、当前维护目标。
- [ ] 明确甲方 / 乙方需求。
- [ ] 明确上下游系统和兼容边界。
- [ ] 明确当前部署环境和运行条件。

## 审计阶段

- [ ] 已读取 README、启动文档、构建配置、部署配置。
- [ ] 已确认技术栈和版本。
- [ ] 已确认数据库、中间件、外部 API。
- [ ] 已确认核心流程。
- [ ] 已查看 `git status` 和现有改动。
- [ ] 已做大规模代码审查。
- [ ] 已记录 P0/P1/P2/P3 问题。
- [ ] 已识别历史严重事故或重复缺陷。
- [ ] 已建立技术债基线。
- [ ] 已建立质量指标基线。
- [ ] 已识别高影响变更，必要时建立变更影响评估。

## 修复阶段

- [ ] P0 问题已修复或有规避方案。
- [ ] P1 问题已修复或排期。
- [ ] 修复没有改变对外兼容边界。
- [ ] 如必须改变兼容边界，已完成人工确认和变更影响评估。
- [ ] 已补充回归测试或替代验证。
- [ ] 已更新缺陷修复计划。

## 后续开发前

- [ ] 既有项目接手审计已完成。
- [ ] 缺陷修复计划已完成第一版。
- [ ] 版本路线图已更新。
- [ ] 技术债记录已更新。
- [ ] 质量指标记录已更新。
- [ ] 严重历史问题已进入事故复盘或缺陷复盘。
- [ ] review/refactor 中版本已规划。
- [ ] 人工确认允许进入后续功能开发。
