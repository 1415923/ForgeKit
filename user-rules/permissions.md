# 用户级规则：权限策略

用于定义 Codex 在本机执行命令和修改文件时的权限边界。

详细命令分级见：

- `user-rules/command-policy.md`

## 默认允许

- 读取当前工作区内的文件。
- 在当前工作区内创建、修改项目文件。
- 运行只读查询命令，例如：
  - `Get-ChildItem`
  - `rg`
  - `git status`
  - `git diff`
- 读取项目内或用户明确给出的环境说明文档。
- 检查本机工具是否存在，例如 `Get-Command`、`java -version`、`node --version`。
- 检查服务状态和端口连通性，例如 `Get-Service`、`Test-NetConnection`。
- 运行项目内明确记录的测试、构建、格式化命令。

## 必须先确认

- 安装依赖或下载外部资源。
- 修改系统环境变量。
- 修改工作区之外的文件。
- 启动需要长时间运行的服务。
- 执行数据库迁移、清库、重建索引等有数据影响的操作。
- 执行 `git commit`、`git push`、`git tag`、发布包、部署等交付动作。
- 修改第三方服务、云资源、远程仓库、issue、PR、邮件、消息、社交媒体或 Webhook。
- 启用新的 MCP、agent、skill、hook 或插件，尤其是需要网络、凭据或工作区外访问的组件。
- 启动长期运行服务，例如 MySQL、Redis、MinIO、Kafka、Elasticsearch、Ollama、Spring Boot 后端、前端 dev server。
- 运行需要正常本机权限或可见终端的命令。

## 默认禁止

- 未经确认删除大量文件。
- 未经确认执行 `git reset --hard`、强制切换分支、强制推送。
- 未经确认清理用户目录、系统目录、缓存目录。
- 为了绕过权限限制而改用其他脚本或工具。

## 破坏性操作规则

任何可能导致数据丢失、历史丢失、远程状态变化的操作，都必须先说明：

- 要执行的命令。
- 会影响哪些文件、目录、分支、数据库或远程资源。
- 是否可恢复。
- 不执行该操作的替代方案。

## 外部动作边界

网络工具默认按只读处理。可以搜索、读取、分析和生成本地草稿，但以下动作必须先确认：

- 发布、推送、合并、部署。
- 修改第三方资源。
- 使用会产生费用的 API。
- 发送外部消息。
- 写入或更新凭据。

如果用户确认不明确，先输出本地计划、草稿或建议命令，不实际执行外部动作。

## 本机权限经验

- Git 查询可以在沙盒中执行；`git push` 通常需要升级权限或正常本机权限。
- Windows PowerShell 原生命令可能出现中文乱码；需要准确读取中文文档时优先使用 PowerShell 7。
- PowerShell 7 适合读写中文文档，但部分命令可能遇到权限或兼容问题；失败后可改用 Windows PowerShell、CMD 或请求升级权限。
- MinIO、Kafka、Elasticsearch、Ollama、Vite dev server 等长期服务不宜在受限沙盒中反复尝试启动。
- 服务启动优先参考项目自己的 `.codex/commands.md`、`README.md` 或专项环境启动指南。
