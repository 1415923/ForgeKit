# 项目初始化问答

用于新项目开始时收集必要信息。回答后，Codex 应只加载相关技术栈模板，并据此补齐 `.codex/` 和 `docs/`。

| 问题 | 回答 |
| --- | --- |

| 项目一句话目标是什么？ | 待补充 |
| 项目类型是什么？例如后端、前端、全栈、脚本、FPGA、桌面工具 | 待补充 |
| 目标用户是谁？ | 待补充 |
| 最终交付形式是什么？例如源码、可执行文件、Web 服务、Docker 镜像、bitstream、PPT | 待补充 |
| 当前阶段是什么？例如原型、MVP、开发、维护、复现 | 待补充 |
| 当前版本或里程碑是什么？ | 待补充 |
| 这次初始化希望 Codex 做到什么程度？ | 待补充 |

## 3. 技术栈选择

只选择当前项目需要的模板。

| 模板 | 是否使用 | 说明 |
| --- | --- | --- |
| `java-springboot` | 否 | Java 后端、Spring Boot |
| `vue` | 否 | Vue / Vite 前端 |
| `react` | 否 | React 前端 |
| `python-fastapi` | 否 | Python API / FastAPI |
| `node-express` | 否 | Node.js / Express 后端 |
| `fpga-vivado-vitis` | 否 | FPGA / Vivado / Vitis / HLS |

如果不在上表中，请写明：

```text
待补充
```

## 11. 初始化后 Codex 应输出

初始化完成后，Codex 应输出：

- 项目判断。
- 选择的技术栈模板。
- 已创建或更新的文件。
- 仍需用户确认的问题。
- 建议的第一批开发任务。
- 可以直接运行的验证命令。
- 不能运行或需要权限确认的命令。
