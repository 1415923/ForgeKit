# 安全治理

本文件定义项目安全治理流程，补充 `.codex/security.md` 的执行层规则。

## 适用范围

以下变更必须进入安全治理流程：

- 登录、鉴权、授权、角色、权限。
- Token、Session、Cookie、密码、密钥、证书。
- 用户数据、隐私数据、业务敏感数据。
- 文件上传、下载、预览、解析。
- SQL、搜索、脚本执行、模板渲染。
- 外部 API、Webhook、MCP、第三方服务。
- 依赖升级或新增。
- 部署、环境变量、云资源、网络暴露。
- 硬件接口、板卡、设备控制。

## 安全治理产物

| 产物 | 作用 |
| --- | --- |
| `.forgekit/docs/threat-model.md` | 识别资产、入口、威胁和缓解措施 |
| `.forgekit/docs/dependency-review.md` | 审查新增或升级依赖 |
| `.forgekit/docs/change-impact.md` | 记录安全相关高影响变更 |
| `.forgekit/docs/risk-register.md` | 记录未关闭安全风险 |
| `.forgekit/docs/incident-review.md` | 记录安全事故 |

## 安全检查层级

| 层级 | 触发条件 | 要求 |
| --- | --- | --- |
| S0 | 不涉及安全敏感面 | 按普通检查 |
| S1 | 涉及输入、配置、依赖、外部 API | 做安全检查清单 |
| S2 | 涉及鉴权、权限、密钥、用户数据 | 做威胁建模和人工确认 |
| S3 | 涉及生产数据、安全事故、密钥泄露 | 阻塞发布，必须复盘 |

## Codex 行为

- 发现安全敏感变更时，先提醒更新安全文档。
- 不输出完整密钥、令牌、密码、私钥。
- 密钥疑似泄露时，提示轮换，而不是只删除文件。
- 新增依赖时，要求说明用途、来源、维护状态和替代方案。
- 发布前检查未关闭 S2/S3 风险时，不建议发布。
