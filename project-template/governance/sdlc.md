# SDLC 通用治理

本文件定义 Codex 参与项目时的标准软件开发生命周期。

## 阶段

| 阶段 | 目标 | 主要产物 | Codex 行为 |
| --- | --- | --- | --- |
| 0. 入口判断 | 判断新项目、接手项目、Bug 修复、重构、发布 | 入口结论 | 选择对应 skill |
| 1. 需求澄清 | 明确目标、用户、范围、非目标 | 需求文档、问答表 | 追问不清楚项 |
| 2. 方案设计 | 确定技术栈、落地条件、风险 | 项目开发方案 | 不急于编码 |
| 3. 架构治理 | 设计模块、接口、数据流、关键决策 | 架构设计、ADR | 记录取舍 |
| 4. 版本规划 | 拆大版本和 review/refactor 中版本 | 版本路线图 | 设置推进闸门 |
| 5. 实现 | 小步实现、保兼容、按范围开发 | 代码、测试 | 遵守技术栈规则 |
| 6. 验证 | 测试、构建、安全检查、人工验证 | 测试文档、验证记录 | 说明未验证项 |
| 7. 发布 | 发布检查、版本记录、回滚 | 版本更新记录 | 不自动 push/deploy |
| 8. 复盘 | review/refactor、文档同步、技术债整理 | 中版本结论 | 不通过则不推进 |

## Codex 工作原则

- 不把“能写代码”当成“可以开始写代码”。
- 初期重点是减少错误方向，而不是快速堆功能。
- 接手项目先保稳定，再谈优化。
- 每个阶段都要有可检查产物。
- 文档不是形式，文档用于让后续开发少重构。
