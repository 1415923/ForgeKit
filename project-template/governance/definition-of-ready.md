# Definition of Ready

本文件定义“什么工作可以进入开发”。

## 功能开发 Ready

- [ ] Feature 或 Task 有编号，例如 `FEAT-001`、`TASK-001`。
- [ ] 需求有编号，例如 `REQ-001`。
- [ ] 目标和非目标明确。
- [ ] 验收标准明确。
- [ ] 影响范围已识别。
- [ ] 技术栈和相关模板已确认。
- [ ] 需要 ADR/RFC 的事项已创建或明确不需要。
- [ ] 测试或验证方式明确。
- [ ] 权限、安全、数据、兼容风险已初步评估。
- [ ] 当前任务状态已进入 `Ready` 或已人工确认。

## Bug 修复 Ready

- [ ] 缺陷有编号，例如 `BUG-001`。
- [ ] 有复现方式或明确现象。
- [ ] 严重程度已判断。
- [ ] 影响范围已判断。
- [ ] 是否触碰兼容边界已判断。
- [ ] 修复后验证方式明确。

## 接手项目后续开发 Ready

- [ ] 既有项目接手审计已完成。
- [ ] P0 已修复或规避。
- [ ] P1 已修复或排期。
- [ ] 兼容边界已记录。
- [ ] 版本路线图已更新。
- [ ] 人工确认允许进入后续功能开发。

## 不 Ready 时 Codex 行为

如果不满足 Ready，Codex 应先追问或补文档，不应直接编码。
