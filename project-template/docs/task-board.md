# 项目任务看板

本文只记录拆解后的执行状态。任务来源和原文在 `.forgekit/docs/task-intake.md`，稳定需求事实在 `.forgekit/docs/requirements.md`，用户可见变化在 `.forgekit/docs/changelog.md`。

## 使用规则

- 每个来自派发内容的任务必须能反链到 `Source ID`。
- 只写当前状态、owner、下一步和验证方式。
- 不复制任务原文，不写长背景，不替代 work log。
- 已完成任务保留结论即可；长过程放 `work-log.md` 或 `.forgekit/changes/<id>/`。

## 状态定义

| 状态 | 说明 |
| --- | --- |
| Backlog | 已记录，未准备开发 |
| Ready | 已满足开始条件 |
| Todo | 已排入当前范围，尚未开始 |
| In Progress | 正在实现 |
| Review | 等待审查或验收 |
| Done | 已完成并通过验证 |
| Blocked | 被阻塞，原因和解除条件已记录 |
| Dropped | 已裁剪或放弃，原因已记录 |
