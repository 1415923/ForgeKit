# 工作日志

本文只记录最近工作窗口，用于交接和中断会话恢复。它不是工作来源台账、不是任务看板、不是全量历史仓库、不是正式发布说明，也不是测试报告。

工作日志必须引用 `Task ID` 或 `Source ID`。日志里的“后续跟进”不会自动变成任务；只有用户确认或满足任务准入规则后，才同步到 `task-board.md`。

## 写入边界

| 写在这里 | 不写在这里 |
| --- | --- |
| 当前阶段按顺序发生了什么 | 工作来源原文 |
| 任务推进过程和验证摘要 | 当前任务总表 |
| 提交 / 推送状态 | 可直接用于 MR 的 changelog |
| 阻塞变化和解除原因 | 风险登记册 |
| 领导或组长确认结论 | 敏感运维数据 |
| 个人后续待办 | 长技术设计 |

## 更新触发

发生以下情况时更新：

- 阶段收口。
- 验证完成。
- 提交或推送完成。
- 阻塞状态变化。
- 领导或组长确认结论。
- 重要排期或责任变化。
- 用户要求或确实需要每日总结。
- 中断会话需要恢复上下文。
- 用户明确要求同步工作日志。

仅稳定技术事实变化时，不强制更新工作日志。

## 对齐规则

- 工作日志可以引用 `Task ID` 和 `Source ID`，但不保存任务原文。
- 工作日志里的待办不得直接进入看板；先判断是否满足任务准入。
- 阻塞解除、领导确认或验证完成后，如果影响任务状态，再同步到 `task-board.md`。
- 用户可见完成结果同步到 `changelog.md`，不要从工作日志直接复制流水。
