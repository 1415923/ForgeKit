# Codex 下一步工作单

用于项目初始化后，把“已确认事实”推进为可讨论、可执行、可验证的下一步。Codex 不应只输出摘要；应根据本工作单继续访谈、查阅项目材料、收敛方案。

## Codex 应执行的下一步

1. 读取 `AGENTS.md`、`.forgekit/docs/project-suitability.md`、`.forgekit/docs/codebase-map.md`、`.forgekit/docs/local-toolchain.md` 和 `.codex/init.generated.md`。
2. 如需确认本机工具链，先运行 `scripts/detect-local-toolchain.ps1`，只记录结果，不自动安装。
3. 输出 3 到 5 个最关键问题，优先确认 MVP 范围、落地环境、验证方式和依赖裁剪。
4. 如果用户答不上来，不要反复追问同一个问题；应给出 2 到 4 个可选方案、取舍、推荐默认值和验证办法。
5. 如果当前上下文不足，应提出查阅路径：本地已有文档、官方文档、同类开源项目、技术选型对比资料或小型验证原型。需要联网时先说明目的并请求确认。
6. 把每个关键项标记为：已确认、暂定假设、需要调研、延期不做。
7. 根据用户回答更新 `.forgekit/docs/project-plan.md`、`.forgekit/docs/tech-decisions.md`、`.forgekit/docs/version-roadmap.md`、`.forgekit/docs/task-board.md` 和 `.forgekit/docs/risk-register.md`。
8. 如果涉及跨模块、高风险或既有项目整改，先更新 `.forgekit/docs/exploration-report.md` 和 `.forgekit/docs/implementation-plan.md`，不要直接编码。

## 方案访谈要求

- 不要把初始化做成一次性填表。Codex 应和用户多轮收敛，直到 v0.1.0 最小闭环、数据边界、部署边界、验证方式和安全边界能自洽。
- 用户只描述业务想法时，先转成：目标用户、核心痛点、MVP 流程、明确不做、数据模型、技术选型候选、风险、第一实现切片。
- 对新项目，方案商讨是独立阶段。先讨论最终产品形态、目标用户、核心功能边界和长期路线，再讨论数据库、登录、目录、框架等工程参数。
- 如果项目方向较宽，应给出 2 到 4 个产品形态候选，并比较功能范围、复杂度、部署成本、长期扩展和不做什么。
- 长期路线图至少要覆盖 v0.1.0、v0.2.0、v0.3.0 和 v1.0.0，每个版本写清目标、验收标准和明确不做。
- 对不确定问题给出推荐默认值，但必须标记为“暂定假设”，不能写成已确认事实。
- 方案未确认前，不要开始大规模编码；可以建议小型验证原型或资料查阅任务。
- 如果公开资料有帮助，应询问用户是否允许联网查阅官方文档、公开项目、同类产品或技术选型对比。未调研时必须明确说明“尚未调研”。
- 如果 `.forgekit/docs/version-roadmap.md` 或 `.forgekit/docs/task-board.md` 仍包含模板示例内容，应先替换为真实项目的 v0.1.0 / v0.1.1 计划，不要保留 ForgeKit 自身历史任务。

## Execution Confirmation / 执行前确认摘要

开始生成业务代码、安装依赖、初始化 Git、commit、push、启动服务、部署或写外部系统前，必须先在会话中输出“执行前确认摘要”，并等待用户明确确认。

摘要至少包括：

- 项目最终目标一句话。
- 已选产品形态，以及放弃的候选方案。
- v0.1.0 / v0.2.0 / v0.3.0 / v1.0.0 版本规划。
- 本次执行做什么、不做什么。
- 技术栈、存储、鉴权、部署和验证命令。
- 参考资料 / 公开项目 / 官方文档调研结论；如果未联网调研，要明确说明。
- 主要风险和安全边界。
- 即将修改或创建的文件范围。
- 即将执行的命令。
- Git、GitHub、部署、服务启动等外部动作。

用户只说“允许生成工程”“允许安装依赖”“数据库用 MySQL”等，只代表局部决策确认，不等于确认完整执行方案。

## 禁止动作

- 不要因为模板已生成就直接开始业务编码。
- 不要在没有执行前确认摘要的情况下生成业务代码、安装依赖、初始化 Git、commit、push 或部署。
- 不要默认安装依赖、启动长期服务、修改数据库、部署、commit、tag 或 push。
- 不要一次性读取全部 `governance/` 和 `docs/`。
- 不要把未确认内容写成事实；未知项保留为待确认。
