## 技术债来源

- 接手项目审计。
- 代码审查。
- 发布检查。
- 版本 review/refactor。
- RFC / ADR 评审。
- 测试和部署验证。

## 技术债分级

| 等级 | 说明 | 处理要求 |
| --- | --- | --- |
| 高 | 阻碍版本推进、影响稳定性、安全或核心架构 | 必须进入近期版本或风险登记册 |
| 中 | 影响维护效率或后续扩展 | 安排到 review/refactor 中版本 |
| 低 | 局部质量问题 | 记录并择机处理 |

## 版本复盘

每个 review/refactor 中版本结束时，记录：

- 新增技术债：待补充
- 已偿还技术债：待补充
- 接受不处理技术债：待补充
- 转为风险项：待补充
