## Codex 应执行的下一步

1. 读取 `AGENTS.md`、`docs/项目适用性评估.md`、`docs/代码库地图.md`、`docs/本地工具链检查.md` 和 `.codex/init.generated.md`。
2. 如需确认本机工具链，先运行 `scripts/detect-local-toolchain.ps1`，只记录结果，不自动安装。
3. 输出 3 到 5 个最关键问题，优先确认 MVP 范围、落地环境、验证方式和依赖裁剪。
4. 根据用户回答更新 `docs/项目开发方案.md`、`docs/技术选型.md`、`docs/版本路线图.md`、`docs/项目任务看板.md` 和 `docs/风险登记册.md`。
5. 如果涉及跨模块、高风险或既有项目整改，先更新 `docs/探索报告.md` 和 `docs/实施计划.md`，不要直接编码。

## 禁止动作

- 不要因为模板已生成就直接开始业务编码。
- 不要默认安装依赖、启动长期服务、修改数据库、部署、commit、tag 或 push。
- 不要一次性读取全部 `governance/` 和 `docs/`。
- 不要把未确认内容写成事实；未知项保留为待确认。
