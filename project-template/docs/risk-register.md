## 使用规则

- 只有仍影响交付、安全、兼容、成本或排期的风险保留在开放风险中。
- 普通 Bug 写 `task-board.md`；严重或重复缺陷才写 `defect-review.md` 或 `incident-review.md`。
- 风险状态变化时同步关联任务或 change，而不是复制长段过程。
