# ForgeKit First-Principles Pass

Use for unclear root causes, repeated patches, high-risk design, architecture, data flow, permissions, security, migration, upgrade, or performance work.

Do not use for typos or explicit low-risk local edits.

Output exactly these sections:

1. Problem / Goal
2. Known Facts
3. Assumptions
4. Constraints
5. First-Principles Derivation
6. Surface Fix vs Root Fix
7. Minimal Verifiable Outcome
8. Recommended Next Step

Separate evidence-backed facts from assumptions. Do not write unverified derivations as facts. Do not edit code. Return implementation to the maker.
