# Stack Status

No stack template is selected at initialization time by default. This is normal.

For a new project, discuss product goals, users, constraints, risks, and the v0.1.0 closed loop before choosing framework, database, runtime, or deployment shape.

For an existing project, infer the current stack from README files, dependency manifests, build scripts, source roots, tests, and runtime configuration before asking the user technical-stack questions.

Only read or add `.codex/stacks/<stack>/` after the stack is confirmed by project evidence or by an explicit planning decision.
