# 缺陷修复计划

用于接手已有项目后的 bug 检查和修复阶段。目标是先修复明确问题，保持兼容，不做大架构改动。

## 修复原则

- 优先修复可复现、影响明确的问题。
- 默认不改变大架构和外部协议。
- 修复前记录复现方式。
- 修复后补充测试或替代验证。
- 涉及兼容边界时先人工确认。
