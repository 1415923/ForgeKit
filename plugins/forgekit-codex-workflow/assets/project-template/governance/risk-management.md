# 风险管理流程

本文件定义风险识别、登记、缓解和关闭流程。

## 风险来源

- 项目开发方案。
- 技术选型。
- RFC / ADR。
- 接手项目审计。
- 缺陷修复计划。
- 安全审查。
- 发布检查。
- 用户或客户约束。
- 变更影响评估。
- 事故 / 缺陷复盘。
- 安全威胁建模。
- 依赖安全审查。

## 风险处理流程

```text
识别 -> 编号 -> 评估概率和影响 -> 制定缓解措施 -> 跟踪状态 -> 关闭或接受
```

## Codex 行为

- 发现风险时，建议写入 `docs/风险登记册.md`。
- 高风险未处理时，不应建议直接进入下一阶段。
- 如果用户接受风险，应记录接受原因和后续补救计划。
- 发布前检查必须查看高风险项是否关闭或被接受。
- 高风险变更应同步到 `docs/变更影响评估.md`。
- 事故复盘中的未关闭高风险项应同步到 `docs/风险登记册.md`。
- 未关闭 S2/S3 安全风险应同步到 `docs/风险登记册.md`。

## 风险分级

| 等级 | 说明 | 行为 |
| --- | --- | --- |
| 高 | 可能导致交付失败、数据损坏、安全事故、兼容破坏 | 阻塞推进，需人工确认 |
| 中 | 可能影响质量、工期、维护成本 | 需要缓解计划 |
| 低 | 可接受或影响有限 | 记录即可 |
