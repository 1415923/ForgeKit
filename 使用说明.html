Codex CLI 开发流程模板使用说明

按成熟 SDLC、架构治理、版本治理和接手项目治理裁剪成 Codex 可执行流程。

你应该怎么走

先选入口：新项目走“SDLC + RFC/ADR + 方案设计 + 架构治理 + 版本路线图”；接手既有项目走“现状审计 + 缺陷修复 + 兼容边界”，然后再规划后续开发。Ready/Done、风险登记、追踪矩阵、变更影响评估、事故/缺陷复盘、安全治理、技术债、质量指标和版本闸门始终适用。
1 先做方案设计

明确项目目标、技术栈、软硬件落地条件和版本路线图，不清楚的先让 Codex 追问。

2 生成项目骨架

复制 PowerShell 命令执行，创建 `.codex/`、`docs/`、skills、技术栈模板和版本闸门。

3 生成方案问答

把你知道的信息填进去，不知道的留空，让 Codex 继续和你商讨并补齐方案。

4 按版本闸门推进

大版本结束后先做 review/refactor 中版本，不通过就不推进下一个大版本。

Lite

小脚本、小工具、个人验证项目。维护 `.codex/`、项目开发方案、版本更新记录和必要命令。

Standard

普通前后端、API、数据处理、内部业务项目。增加需求、架构、技术选型、版本路线图、任务看板、测试和发布检查。

Enterprise

多人协作、接手项目、交付项目、高风险项目。增加 ADR/RFC、风险、变更影响、代码所有权、CI/CD、安全治理和复盘。

浏览器不能直接执行 PowerShell，也不能稳定获取 Windows 完整路径。本页负责生成命令、问答和提示词，执行动作仍在 PowerShell 或 Codex 中完成。

1. 项目模板初始化

选择文件夹后，页面会把“常用父目录 + 文件夹名”拼成完整目标路径；如果实际路径不同，请手动修改。

复制到 PowerShell 执行

执行成功后，到新项目目录打开 Codex，再继续下面的问答和初始化提示词。

常见组合

普通 Java 前后端项目：选 java-springboot + vue。
只有 Java 后端：只选 java-springboot。
只有前端：Vue 项目选 vue，React 项目选 react。
FPGA / HLS 项目：只选 fpga-vivado-vitis，除非项目确实还有软件端。

2. 方案设计与项目初始化问答

这部分是最重要的。你可以先填你知道的内容，空着的让 Codex 继续追问和讨论；没有形成方案前，不建议直接编码。

生成的问答内容

你可以把这段发给 Codex，让它写入 .codex/questionnaires/项目初始化问答.md 并继续补齐项目规则。

3. 给 Codex 的下一步提示词

先执行初始化命令，再复制下面提示词到新项目目录的 Codex 会话中。

本地模板入口

project-template templates scripts questionnaires skills

生成后目录说明

目录或文件用途
.codex/项目目标、范围、命令、风格、测试、安全、Git、skills 规则。
.codex/stacks/<stack>/按需复制的技术栈模板，避免加载无关规则。
.codex/questionnaires/项目初始化问答表，作为补齐文档和规则的事实来源。
.agents/skills/项目级 Codex skills：project-init、code-review、release-check、security-review。
docs/需求、架构、技术选型、接口、数据库、部署、测试、版本记录。
