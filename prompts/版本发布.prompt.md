# 版本发布 Prompt

请协助准备版本发布。

要求：

1. 读取 `docs/版本更新记录.md`、`.codex/git.md`、`.codex/commands.md`、`docs/环境矩阵.md`、`docs/发布流水线.md`、`docs/部署文档.md`、`docs/代码所有权.md`、`docs/项目任务看板.md`。
2. 查看当前 Git 状态。
3. 汇总本版本新增、变更、修复、移除内容。
4. 检查测试、构建、文档、配置、接口、数据库迁移、目标环境、制品、部署步骤、回滚方案、必要评审人和当前版本任务状态是否完成。
5. 给出发布前风险清单。
6. 不要自动 commit、tag、push 或部署，除非我明确要求。

输出格式：

- 发布摘要
- 检查结果
- 风险和阻塞项
- 环境和发布流水线检查
- 代码所有权和评审责任检查
- 项目任务状态检查
- 建议执行命令
- 是否可以发布
