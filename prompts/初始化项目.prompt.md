# 初始化项目 Prompt

请基于当前项目目录，帮我完成项目初始化。

要求：

1. 先读取当前目录结构、README、构建配置和已有文档。
2. 先读取 `governance/流程总览.md`，判断项目入口。
3. 如果存在 `.codex/stacks/`，只读取与当前项目相关的技术栈模板。
4. 如果存在 `.codex/questionnaires/项目初始化问答.md` 或其他项目初始化问答表，优先读取并根据答案补齐项目事实。
5. 判断项目类型、技术栈、运行方式和当前状态。
6. 先确认项目开发方案、技术栈选择、软硬件落地条件、环境矩阵、发布流水线、代码所有权、项目任务模型、版本路线图和版本推进闸门。
7. 根据项目实际技术栈，只选择需要的 `templates/<stack>/`，不要加载无关技术栈模板。
8. 如果信息不足，先列出需要我补充的问题，不要直接进入编码。
9. 补齐或创建以下内容：
   - `.codex/project.md`
   - `.codex/scope.md`
   - `.codex/commands.md`
   - `.codex/style.md`
   - `.codex/testing.md`
   - `.codex/security.md`
   - `.codex/version-gates.md`
   - `.codex/git.md`
   - `docs/项目开发方案.md`
   - `docs/版本路线图.md`
   - `docs/需求文档.md`
   - `docs/架构设计.md`
   - `docs/技术选型.md`
   - `docs/环境矩阵.md`
   - `docs/发布流水线.md`
   - `docs/代码所有权.md`
   - `docs/项目任务看板.md`
10. 将选中的技术栈模板内容合并到项目规则或保留为独立说明。
11. 不要修改业务代码，除非我明确要求。

输出格式：

- 项目判断
- 项目开发方案是否足够进入编码
- 选择的技术栈模板
- 已补齐内容
- 仍需确认的问题
- 建议的下一步
